# Cognitive V3 Findings — First Learned Module (Source/Authority Classifier)

V3 is the bridge the roadmap named: **raw documents in → trusted resolution out,
with no manually supplied authority labels**. The V2 contract *assumed* every
evidence item's authority was given as ground-truth metadata. V3 removes that
assumption for the authority field by *learning* it, and measures the honest cost.

Branch `research/cognitive-contract-v1`. Baseline frozen at tag
`v0.2-cognitive-contract` (commit d712c36). This is a from-scratch model on
synthetic data, CPU-only — a credible baseline, not a frontier result.

## What was built

- `nueronce/authority_data.py` — taxonomy, provenance→authority ground-truth rule, and
  a randomized generator of labeled documents (clean, hard-negative, and three
  tiers of channel-spoofing forgeries).
- `nueronce/authority_clf.py` — a small from-scratch classifier: hashed char-n-gram +
  provenance-metadata features → tiny MLP (torch as tensor substrate only), with
  temperature-scaled calibration and confidence abstention.
- `nueronce/authority_eval.py` + `scripts/train_authority.py` — metrics and the
  end-to-end error-propagation harness that feeds *predicted* authority into the
  V2 deterministic policy.
- Independent **blind** set `data/authority_blind.jsonl` (400 lines) generated by a
  separate process (`nueronce/authority_redteam.py`) with its own logic — a held-out
  test the classifier's training never saw.

## Results (seed 0)

Classifier:

| set | n | accuracy | impersonation acceptance | false-rejection | ECE |
|---|---|---|---|---|---|
| held-out | 1200 | 0.994 | 0.009 | 0.006 | 0.004 |
| blind (independent) | 400 | 0.950 | 0.000 | 0.050 | 0.022 |

Impersonation acceptance by attack family (held-out) — the core result:

| attack family | acceptance | note |
|---|---|---|
| impersonation ("SYSTEM MESSAGE") | 0.000 | resisted |
| authority laundering | 0.000 | resisted |
| citation spoofing | 0.000 | resisted |
| paraphrased official | 0.000 | resisted |
| unicode confusable | 0.000 | resisted |
| metadata/text conflict | 0.000 | resisted |
| spoof_crude / spoof_medium | 0.000 | resisted |
| **spoof_perfect** | **0.278** | **information-theoretic ceiling** |

End-to-end error propagation (predicted vs oracle authority into the V2 policy):

| authority source | resolution accuracy | poison rate |
|---|---|---|
| oracle (ground-truth metadata) | 1.000 | 0.000 |
| predicted (learned classifier) | 0.838 | 0.070 |
| **cost of learning** | **−0.162** | **+0.070** |

## What this establishes (honestly)

1. **All text-based attacks are resisted (0.000 acceptance).** Because authority is
   inferred from provenance features (channel/domain/signature), not from claims in
   the text, impersonation, laundering, citation spoofing, paraphrase, and unicode
   attacks fail — including on the independently-generated blind set.
2. **Perfect metadata forgeries cannot be resisted by any text/metadata classifier.**
   `spoof_perfect` documents are *feature-identical* to genuine trusted records
   (trusted channel + real gov/corp domain + signed + plain official text); nothing
   observable separates them, so acceptance sits near the base rate (0.278). This is
   a property of the information available, not a modeling deficiency. The fix is
   **cryptographic provenance** — signed documents and hash verification — which is
   **Phase 1** ("add signed provenance", "verify document hashes"), not a better
   classifier.
3. **Per-document accuracy does not guarantee end-to-end correctness.** A 0.994
   classifier yields only 0.838 end-to-end resolution accuracy and a 0.070 poison
   rate, because errors land on the *decisive* documents (the trusted correction and
   the poison). Rare misclassifications are amplified by resolution. This is the
   central lesson for Phase 7 integration: measure error propagation, gate
   low-confidence labels, and add human-review thresholds.

## What this does NOT establish

- Real-world performance. Data is synthetic; the blind set is independently
  generated but still synthetic and single-author. Real external/human-created
  documents remain **Phase 2** work.
- That this is the best model. Only a small from-scratch baseline was trained; the
  roadmap's "compare rules, embeddings, transformers, hybrid" comparison is not
  done.
- Generalization to genuinely unseen organizations at scale, or source-history /
  link / signature-verification features (listed Phase-3 inputs not yet used).

## Next (per roadmap "following stage")

1. **Phase 1 hardening first for the perfect-forgery ceiling**: add signed
   provenance + document-hash verification so authority does not rest on spoofable
   metadata alone.
2. **Phase 4 claim extraction** and **Phase 5 temporal/scope/revocation extraction**
   — the remaining learned modules needed for full "raw documents → trusted claims".
3. **Phase 2 external benchmark** — replace synthetic evaluation with real/blind
   human-created documents before making any capability claim.
