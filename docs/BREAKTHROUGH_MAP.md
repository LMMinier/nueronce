# CFNA Breakthrough Map — connecting the dots across every branch

*An analysis pass over the full project state (design doc, all reports, all
branches) answering three questions: what is actually proven, where is the
work genuinely close to a breakthrough, and where is the current framing
blind to its own results. Written 2026-07-01 against `claude/new-session-tkjr3b`
lineage; benchmark reproductions in this doc were re-run on this branch.*

---

## 1. The hypothesis ledger — H1..H8 against the evidence

| Hyp | Claim | Evidence state |
|---|---|---|
| H1 | Dynamic byte patches cut compute while preserving exactness | Mechanism real (learned boundary head, ~7-byte patches, non-degenerate); the compute claim untested — attention still computes dense-then-mask |
| H2 | Typed recurrent state improves long-trajectory stability | **Tested (Wave 3) — unsupported at this scale.** Channel-ablation probe (`benchmarks/h2_channel_probe.json`): removing *all* typed channels raises loss only on the long-range recall task (~3.5%) and near-zero elsewhere; removing any *single* channel changes loss <0.3% for every task. The channels are a diffuse redundant pool, not specialized roles — the dense read-out redistributes. H2's specialization claim is falsified at this scale/training; see §3.3 |
| H3 | Dense+sparse+late retrieval improves citation precision | Retrieval is *load-bearing* (0.10→~0.7 accuracy in the RETRO-style ablation) — but that proves the retrieval **path**, not the three-way fusion vs dense-only |
| H4 | Planner-before-renderer improves completeness | Not measured |
| H5 | Independent verification reduces unsupported claims | Supported (V3.3 ablation: unsupported 0.090 → 0.000), heuristic verifier |
| H6 | Authority masks reduce prompt-injection failures | **Strongly supported — the project's best result** (§2) |
| H7 | Multi-objective WPGCP beats causal-only at equal compute | Not started |
| H8 | Residual correction experts beat full fine-tunes | Not started (stubs) |

Honest score: one hypothesis strongly supported, two partially, the central
one (H2) untested, three unstarted. This is not a criticism — it's a map of
where the finite experiment budget went. It went to H6. And H6 paid.

## 2. The near-breakthrough being undersold: the authority/provenance stack

The reports frame FULL=1.000 defensively (correctly: policy ≡ spec is
circular). But the circularity caveat applies to the *absolute* number, not
to the **contrast and adversarial metrics**, which are non-circular and are
the actual result:

- A from-scratch, calibrated (ECE 0.004) authority classifier that holds
  **0.000 acceptance across every text-borne attack family** — impersonation,
  laundering, citation-spoofing, paraphrase, Unicode confusables — on a
  **blind set generated by an independent codebase** (`authority_redteam.py`
  shares no code with the training generator). Baselines poison at 0.75–1.00.
- The single residual failure (`spoof_perfect`, 0.278) was **proven to be an
  information-theoretic ceiling** — feature-identical forgeries are
  unlearnable by construction — and then **eliminated by building the thing
  the proof demands**: Ed25519 provenance (V3.1–V3.3), driving poison
  acceptance to 0.000 with calibrated abstention instead of silent failure.
- The honest cost of removing the labels-given assumption was *measured*,
  not assumed: oracle 1.000 → predicted 0.838 end-to-end, because a 0.994
  classifier's rare errors land on decisive documents. Few systems publish
  this number at all.
- New as of this branch: the V3.2 benchmark JSON reproduces **bit-identical**
  across OS (Windows→Linux), Python (3.13→3.11), and backend (PyTorch→
  microtorch). That is an unusually strong reproducibility property for a
  claimed defense result.

**Framed properly, this is a publishable-shape result:** *a provenance-gated
cognitive contract that provably cannot be poisoned by any text-only attack,
with the residual attack surface reduced to key compromise (and revocation
measured).* What separates it from a defensible external claim is exactly one
artifact, already specified in `docs/PROVENANCE_BLIND_SET_GUIDE.md`: an
externally-authored naturalistic blind benchmark (Phase 2). Not more
mechanism. One dataset.

## 3. The blindness points

### 3.1 The V3.3 "tie" is not a negative result about the architecture
`provenance_v33.py`'s "full_retrieval_resolution_renderer_verifier" system is
a **simulation**: it never calls `pipeline.respond()`, never runs the byte
model, and shares its resolution code path with `provenance_contract`. The
0.920 = 0.920 tie was therefore **guaranteed by construction** — the gate
`full_beats_contract_only: false` measured that a system equals itself, not
that retrieval/rendering/verification add nothing. The recorded conclusion
("retrieval/rendering/verification are not yet contributing") is the right
observation with the wrong implied cause. The remedy is Wave 2 (§5): put the
*real* learned pipeline in the loop and measure again.

### 3.2 Two real systems, zero integration — and the roadmap predicted this
`KNOWN_LIMITATIONS.md` already admits it: the learned byte core and the
symbolic cognitive layer are islands. The only bridge is a data contract
(`ingestion → SourceRecord → respond()`'s evidence gate). The neural model
never conditions on authority; the authority classifier never feeds
`respond()`. Meanwhile `CFNA_concurrent_plan.txt`'s FINAL DIRECTIVE — "do not
spend the next sprint adding more cognitive modules" — was written before
V2→V3→V3.1→V3.2→V3.3, which are all cognitive modules. The drift produced
the strongest result (§2), so it wasn't wasted — but the integration debt is
now the single blocking item between "two good systems" and "one defensible
architecture."

### 3.3 H2 is the thesis — and Wave 3 falsified it at this scale
Every differentiation argument for CFNA over a plain decoder ultimately
leans on typed state. Lane G has now run (`scripts/probe_typed_channels.py`):
train one model on tasks with distinct memory demands (local structure,
long-range key recall, running count), then zero each typed channel and
measure per-task loss increase. Result: the typed memory as a whole is
load-bearing only for long-range recall (~3.5%), and **no single channel
matters (<0.3% for every task)** — the channels are a redundant pool the
dense read-out routes around, not specialized roles. Per the concurrent
plan's own rule, H2 is recorded as unsupported at this scale. This does not
kill the idea: it says specialization must be *forced* (per-channel read-out
heads, channel-specific auxiliary losses, or the WPGCP typed objectives of
H7) rather than hoped for — a concrete redesign target, not a dead end. The
honest consequence for now: **the differentiation story cannot lean on typed
channels until specialization is engineered and re-probed.** What it *can*
lean on is the provenance/authority result (§2), which does not depend on H2
at all.

### 3.4 The renderer's known weakness will be blamed on the wrong layer
The 100K SFT report proved the micro renderer generalizes *structure* but
not *content* (arithmetic 2–5% even on memorized pairs). When the integrated
loop runs, raw answer accuracy will partly reflect renderer fidelity, not
resolution quality. Any integrated evaluation must therefore report
**renderer_fidelity separately from resolution accuracy** — otherwise a
symbolic-layer success will be misread as failure because a 112K-parameter
renderer garbled the final string (the inverse of blindness point 3.1).

## 4. What is already strong enough to state plainly

1. Every neural operator is hand-built, trains, and is verified causal —
   twice (PyTorch and a from-scratch NumPy autograd engine), with
   finite-difference gradient checks and torch-parity at ~1e-8.
2. Retrieval is load-bearing, proven by ablation on facts that cannot be
   memorized.
3. The cognitive contract beats every naive baseline on poisoning by
   construction *and* survives an independently-generated adversarial blind
   set after learning replaces one oracle assumption.
4. Text-only attacks cannot poison the gated system; the residual surface is
   key compromise, and revocation is measured.
5. Negative results are recorded with unusual honesty (V3.3 gate failures,
   SFT arithmetic, held-out baseline ties) — this is the project's real
   moat: the evidence trail is trustworthy.

## 5. The falsification path (what this branch is building)

- **Wave 1 (done, this branch):** whole symbolic stack runs without torch
  (microtorch authority backend, bit-identical V3.2 reproduction); evidence
  gate extracted torch-free; V3.3 negative result replicated on a fresh seed.
- **Wave 2 (in progress):** the first *real* end-to-end learned cognitive
  loop — Ed25519 ingestion → learned authority classifier on unsigned text →
  gated evidence → **actual trained byte model as renderer** → independent
  verifier → answer scored strictly from rendered text, with counterfactual
  citation attribution (cite a document iff removing it flips the outcome —
  directly targeting the failed `citations_identify_decisive_evidence`
  gate). Reported: integrated-vs-contract-only, oracle-vs-predicted gap at
  pipeline level, renderer fidelity as its own number.
- **Wave 3:** Lane G — the H2 specialization probes, run to whatever verdict
  the data gives.
- **Phase 2 (needs the author):** the external naturalistic blind benchmark
  per the existing authoring guide — the one artifact between §2 and a
  claim that survives outside review.

*Everything above stays subject to the concurrent plan's own rule: when a
hypothesis fails, record it and remove it.*
